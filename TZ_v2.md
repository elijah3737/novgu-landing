# Техническое задание v2

## Интерактивный обучающий лендинг для приёмной комиссии НовГУ

> **Версия 2** — переработка исходного ТЗ с акцентом на измеримость, поведенческий результат и продуктовую логику.

---

## 1. Что изменилось по сравнению с v1

| Блок | Было (v1) | Стало (v2) |
|------|-----------|------------|
| Цель | «Сотрудник понимает» | Поведенческие KPI + аттестация |
| Структура | 11 линейных экранов | Меню + фильтр по роли + короткая/полная версия |
| Тренажёр | 4 ситуации × «правильно/неправильно» | 8 ситуаций × ветвление × 5 критериев × оценка |
| Контент | Только текстовые карточки | + банк возражений, этическая рамка, аудио (опц.) |
| Результат | «Понял материал» | Сертификат с именем, PDF-карточка на стол, шаблон CRM-карты |
| Аналитика | Отсутствует | Я.Метрика + события воронки |
| Обновления | Не предусмотрены | Контент вынесен в JSON, обновляется ежегодно |

## 2. Цели и KPI

### Бизнес-цель
Увеличить долю обращений в приёмную комиссию НовГУ, которые доходят до подачи документов, за счёт качественного первого контакта.

### KPI лендинга (продуктовые)
- 90% сотрудников ПК прошли материал до аттестации к 01.06 каждого года
- Средний балл аттестации ≥ 8 из 10
- Время на полное прохождение — ≤ 30 минут
- Доля досматриваемости (scroll depth до экрана «Финал») ≥ 70%

### KPI поведенческие (замеряются в CRM ПК через 2 недели после прохождения)
- Доля звонков с заполненными обязательными полями (имя, телефон, направление, экзамены) — рост от baseline
- Доля «возвращённых» абитуриентов (повторный контакт по инициативе сотрудника) — рост от baseline
- Конверсия «обращение → подача документов» — рост от baseline

> **Baseline нужно зафиксировать до запуска лендинга**, иначе нечего сравнивать.

## 3. Архитектура

### Точки входа
1. Прямая ссылка для самостоятельного изучения
2. Встроенная в корпоративную базу знаний
3. Открытие в режиме «киоск» на вводных встречах

### Сегментация по роли (экран 0)
- Горячая линия / call-центр
- Очная приёмная комиссия (стол)
- Представитель института
- Соцсети и мессенджеры (ВК, Telegram)

→ В зависимости от выбора подсвечиваются релевантные сценарии в тренажёре и адаптируются примеры фраз.

### Две версии прохождения
- **Экспресс 5 минут** — воронка + 7 шагов + чек-лист + 1 ситуация тренажёра + аттестация
- **Полная 30 минут** — все блоки

### Меню навигации
Зафиксировано сверху, позволяет переходить к любому разделу. Прогресс прохождения сохраняется в localStorage.

## 4. Структура лендинга (10 экранов)

### Экран 0. Выбор роли
Сегментация на старте. Один клик — фильтр контента.

### Экран 1. Hero + ключевая цифра
- Заголовок: «От консультации к поступлению»
- 1 ключевая цифра (например: «72% абитуриентов выбирают вуз после первого живого контакта»)
- CTA: «Начать», «Экспресс 5 мин», «Полная 30 мин»

### Экран 2. Воронка абитуриента
Интерактивная воронка из 8 этапов. Клик по этапу → раскрытие смысла этапа и роли сотрудника.

### Экран 3. 7 шагов правильного разговора
Слайдер/аккордеон с примерами фраз для каждой роли.

### Экран 4. Банк вопросов и возражений
5 вкладок:
- Направление
- Экзамены и баллы
- Выбор вуза
- Сомнения
- Возражения (новое — 10 типовых: «у вас низкие рейтинги», «лучше уехать», «работы не будет», «дорого», «далеко» и т.д.)

### Экран 5. Тренажёр (ядро продукта)
- 8 ситуаций (минимум)
- Каждая — 2–3 хода диалога с ветвлением
- 3 варианта ответа на каждом ходе: **плохо / средне / хорошо**
- После каждого выбора — разбор, почему
- Финальная оценка по 5 критериям:
  1. Уточнил ситуацию
  2. Снял возражение
  3. Собрал данные
  4. Зафиксировал контакт
  5. Договорился о следующем шаге

### Экран 6. Этическая рамка
«Сопровождение vs давление» — где граница. Снимает внутреннее сопротивление сотрудников.

### Экран 7. Чек-лист + ассеты для скачивания
- Интерактивный чек-лист
- PDF-карточка на стол (А5, 7 шагов + банк вопросов)
- Шаблон CRM-карты абитуриента
- Скрипт-памятка для горячей линии

### Экран 8. Аттестация
- 10 вопросов с выбором ответа
- Минимум 8/10 для прохождения
- При успехе — сертификат с именем (PDF), генерируется на клиенте

### Экран 9. Финал + следующие шаги
- Главная мысль
- Кнопки: пройти ещё раз, скачать ассеты, перейти к CRM
- Контакт ответственного за обновление материала

## 5. Тренажёр — детально

### Структура ситуации (JSON)
```json
{
  "id": "s1",
  "title": "Не знает баллы",
  "role": ["hotline", "office"],
  "opening": "Я пока не знаю, сколько баллов наберу...",
  "turns": [
    {
      "options": [
        {"text": "Когда узнаете — обращайтесь", "score": "bad", "explain": "Закрывает диалог, теряет контакт"},
        {"text": "Без баллов сложно сказать", "score": "mid", "explain": "Не закрывает, но и не помогает"},
        {"text": "Давайте посмотрим направления. Какие экзамены сдаёте?", "score": "good", "explain": "Переводит к выявлению потребности"}
      ],
      "nextReply": {
        "good": "Я сдаю русский, математику и информатику",
        "mid": "Ну... не знаю...",
        "bad": "[диалог закончен]"
      }
    }
  ],
  "criteria": ["clarified", "objection", "data", "contact", "next_step"]
}
```

### Минимум 8 ситуаций
1. Не знает баллы
2. Выбирает между НовГУ и СПб
3. Родитель вместо абитуриента
4. Боится не пройти на бюджет
5. Возражение: «у вас низкие рейтинги»
6. Возражение: «лучше уехать в столицу»
7. Из другого региона, спрашивает про общежитие
8. Выпускник колледжа, поступление по среднему

## 6. Аналитика

Я.Метрика + следующие события:
- `landing_open` (с UTM)
- `role_selected` (значение роли)
- `version_chosen` (express/full)
- `funnel_step_clicked` (id этапа)
- `simulator_situation_started` (id ситуации)
- `simulator_turn_choice` (id ситуации, ход, оценка)
- `simulator_completed` (id ситуации, итоговый балл)
- `checklist_completed`
- `asset_downloaded` (тип ассета)
- `attestation_started`
- `attestation_passed` (балл)
- `attestation_failed` (балл)
- `scroll_depth` (25/50/75/100%)

## 7. Печатные ассеты (в скоупе!)

### PDF-карточка на стол (А5, двусторонняя)
- Сторона 1: 7 шагов разговора с короткими фразами
- Сторона 2: банк вопросов по 5 темам + топ-5 возражений

### Шаблон CRM-карты абитуриента
- Обязательные поля: имя, телефон, направление, экзамены
- Опциональные: ожидаемые баллы, другие вузы, сомнения, удобное время

### Сертификат
- Имя сотрудника, дата, балл
- Подпись начальника ПК (макет)
- Генерируется на клиенте (JS + jsPDF или просто HTML-печать)

## 8. Технические требования

- Стек: HTML + CSS + JavaScript (без фреймворков)
- Один `index.html` со встроенными CSS/JS (по требованию заказчика — «открывается локально»)
- Адаптив: десктоп / планшет / смартфон
- Прогресс — в localStorage
- Контент тренажёра, банка возражений и аттестации — в отдельном JS-объекте (легко обновлять)
- Все блоки прокомментированы в коде
- Иконки — inline SVG
- Анимации — CSS (без JS-библиотек анимаций)

### Брендинг
- Фирменные цвета НовГУ (тёмно-синий + акцентный)
- Светлый фон, мягкие тени, крупная типографика
- Без перегруза текстом — короткие абзацы, разбивка экранами

---

## 9. Тональность

- Уважительная, обучающая, без критики сотрудников
- «Типовые ловушки» вместо «вы делаете не так»
- «Усиление коммуникации» вместо «продажи университета»

---

## 10. Критерии готовности

- Открывается как `index.html` локально
- Работает на десктопе, планшете, смартфоне
- Все интерактивные элементы функциональны
- Аттестация выдаёт PDF-сертификат с именем
- Скачиваются: PDF-карточка, шаблон CRM-карты
- Прогресс сохраняется между сессиями
- Контент обновляется правкой одного JS-объекта
- Зашита аналитика (заглушки событий, если без Метрики)

---

## 11. Ритм обновления

- Полная ревизия — ежегодно, до 01.04 (старт приёмной кампании)
- Обновление цифр и проходных баллов — ежегодно после официальной публикации
- Дополнения в банк возражений — по обратной связи от сотрудников после первого сезона

---

## 12. Что вне скоупа этой версии

- Интеграция с CRM (только шаблон карты)
- Аудио-записи реальных звонков (требует юридического согласования)
- Видеоразборы (отдельный продукт)
- Мобильное приложение
- Backend / база сертификатов на сервере
